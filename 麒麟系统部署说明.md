# 麒麟系统（Kylin OS）部署说明

## 系统概述

麒麟系统是基于 Linux 的国产操作系统，支持以下架构：
- x86_64（英特尔/AMD）
- ARM64（飞腾、鲲鹏）
- LoongArch（龙芯）

## 部署方式选择

| 方式 | 难度 | 适用场景 | 推荐度 |
|------|------|----------|--------|
| **Docker 部署** | ⭐ 简单 | 有 Docker 环境的麒麟系统 | ⭐⭐⭐⭐⭐ |
| **原生部署** | ⭐⭐ 中等 | 无 Docker 或需要极致性能 | ⭐⭐⭐⭐ |
| **国产数据库适配** | ⭐⭐⭐ 复杂 | 需要使用达梦/人大金仓 | ⭐⭐⭐ |

## 方式一：Docker 部署（推荐）

### 适用版本
- 银河麒麟 V10（SP1 及以上）
- 中标麒麟 V7

### 检查 Docker 支持

```bash
# 检查系统版本
cat /etc/os-release

# 检查是否支持 Docker
uname -m  # 查看架构：x86_64 / aarch64 / loongarch64

# 安装 Docker（如未安装）
sudo apt-get update
sudo apt-get install -y docker.io docker-compose
# 或
sudo yum install -y docker docker-compose

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

### 部署步骤

```bash
# 1. 复制项目到麒麟系统
scp -r storage_system/ user@kylin-server:/home/user/

# 2. 进入项目目录
cd storage_system/

# 3. 运行部署脚本
chmod +x start.sh
./start.sh
```

### 架构适配

如果架构不是 x86_64，需要修改 Docker 镜像：

```yaml
# docker-compose.yml 修改
services:
  backend:
    # 对于 ARM64（飞腾/鲲鹏）
    image: python:3.11-slim-bullseye  # 支持多架构
    
  mysql:
    # 使用支持 ARM64 的 MySQL 镜像
    image: mysql:8.0-oracle  # 支持 ARM64
```

## 方式二：原生部署（无 Docker）

### 安装依赖

```bash
# 更新系统
sudo apt-get update

# 安装 Python 3.11
sudo apt-get install -y software-properties-common
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt-get update
sudo apt-get install -y python3.11 python3.11-venv python3.11-dev python3-pip

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo bash -
sudo apt-get install -y nodejs

# 安装 MySQL（可选）
sudo apt-get install -y mysql-server-8.0

# 安装构建工具
sudo apt-get install -y gcc g++ make libmysqlclient-dev pkg-config
```

### 部署步骤

```bash
# 1. 进入后端目录
cd backend/

# 2. 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 配置数据库等

# 5. 初始化数据库
python -c "
import asyncio
from app.core.database import engine, Base
async def init():
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
asyncio.run(init())
"

# 6. 启动后端
python -m uvicorn app.main:app --host 0.0.0.0 --port 8003 &

# 7. 进入前端目录
cd ../frontend/

# 8. 安装依赖
npm install

# 9. 构建前端
npm run build

# 10. 启动前端（使用 serve）
npm install -g serve
serve -s dist -l 3003
```

## 架构适配说明

### x86_64（标准）

完全兼容，直接使用标准镜像和依赖。

### ARM64（飞腾/鲲鹏）

```bash
# 需要多架构支持的镜像
# docker-compose.yml 修改
services:
  mysql:
    image: mysql:8.0.33-oracle  # Oracle 构建的 ARM64 版本
    
  backend:
    build:
      context: ./backend
      platforms:
        - linux/arm64
```

### LoongArch（龙芯）

龙芯架构需要特殊处理：

```bash
# 1. 使用龙芯提供的 Docker 镜像仓库
# 2. 或者使用源码编译
# 3. Python 和 Node.js 需要从源码编译

# 安装龙芯版本的 Python
wget http://ftp.loongnix.cn/os/loongnix/20/python3/python3_3.8.5-1_amd64.deb
sudo dpkg -i python3_3.8.5-1_amd64.deb
```

## 一键部署脚本

创建 `deploy_kylin.sh`：

```bash
#!/bin/bash

# 麒麟系统一键部署脚本

echo "=================================="
echo "仓储物资出入库动态管理系统"
echo "麒麟系统部署脚本"
echo "=================================="
echo ""

# 检测架构
ARCH=$(uname -m)
echo "检测到的架构: $ARCH"

# 检测系统类型
if [ -f /etc/kylin-release ]; then
    OS="银河麒麟"
    VERSION=$(cat /etc/kylin-release)
elif [ -f /etc/neokylin-release ]; then
    OS="中标麒麟"
    VERSION=$(cat /etc/neokylin-release)
else
    OS="未知系统"
    VERSION="未知版本"
fi
echo "操作系统: $OS $VERSION"
echo ""

# 检查 Docker
if command -v docker &> /dev/null; then
    echo "✓ Docker 已安装"
    USE_DOCKER=true
else
    echo "✗ Docker 未安装"
    echo "将使用原生部署方式"
    USE_DOCKER=false
fi

# 选择部署方式
if [ "$USE_DOCKER" = true ]; then
    read -p "使用 Docker 部署？(Y/n): " use_docker
    if [[ $use_docker =~ ^[Nn]$ ]]; then
        USE_DOCKER=false
    fi
fi

if [ "$USE_DOCKER" = true ]; then
    echo ""
    echo "开始 Docker 部署..."
    
    # 根据架构选择镜像
    if [ "$ARCH" = "aarch64" ]; then
        echo "检测到 ARM64 架构，使用适配镜像..."
        export COMPOSE_FILE=docker-compose.arm64.yml
    elif [ "$ARCH" = "loongarch64" ]; then
        echo "检测到 LoongArch 架构..."
        echo "警告：LoongArch 需要特殊镜像，请确保已准备"
        export COMPOSE_FILE=docker-compose.loongarch.yml
    fi
    
    # 启动服务
    sudo docker-compose up -d
    
else
    echo ""
    echo "开始原生部署..."
    
    # 安装依赖
    echo "安装系统依赖..."
    if command -v apt-get &> /dev/null; then
        sudo apt-get update
        sudo apt-get install -y python3 python3-pip python3-venv nodejs npm
    elif command -v yum &> /dev/null; then
        sudo yum install -y python3 python3-pip nodejs npm
    fi
    
    # 部署后端
    echo "部署后端服务..."
    cd backend
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    
    # 适配国产数据库（如需要）
    read -p "使用国产数据库？(达梦/人大金仓) y/N: " use_native_db
    if [[ $use_native_db =~ ^[Yy]$ ]]; then
        echo "请选择数据库:"
        echo "1. 达梦 (DM8)"
        echo "2. 人大金仓 (KingbaseES)"
        read -p "选择 (1/2): " db_choice
        
        if [ "$db_choice" = "1" ]; then
            pip install dmPython
            sed -i 's/mysql+aiomysql/dm+dmPython/g' app/core/config.py
        elif [ "$db_choice" = "2" ]; then
            pip install psycopg2-binary
            sed -i 's/mysql+aiomysql/postgresql+psycopg2/g' app/core/config.py
        fi
    fi
    
    # 初始化数据库
    python -c "
import asyncio
from app.core.database import engine, Base
async def init():
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
asyncio.run(init())
"
    
    # 启动后端
    nohup python -m uvicorn app.main:app --host 0.0.0.0 --port 8003 > ../logs/backend.log 2>&1 &
    cd ..
    
    # 部署前端
    echo "部署前端服务..."
    cd frontend
    npm install
    npm run build
    nohup npx serve -s dist -l 3003 > ../logs/frontend.log 2>&1 &
    cd ..
fi

echo ""
echo "=================================="
echo "部署完成！"
echo ""
echo "访问地址:"
echo "  - 前端: http://localhost:3003"
echo "  - 后端: http://localhost:8003"
echo ""
echo "默认账号: admin / admin123"
echo "=================================="
```

## 常见问题

### Q: 提示 "docker: command not found"
A: 麒麟系统可能使用 podman 代替 docker：
```bash
sudo apt-get install podman podman-compose
# 使用 podman 代替 docker
podman-compose up -d
```

### Q: Python 版本太低？
A: 麒麟系统默认 Python 3.7，需要升级：
```bash
# 从源码编译 Python 3.11
wget https://www.python.org/ftp/python/3.11.4/Python-3.11.4.tgz
tar xzf Python-3.11.4.tgz
cd Python-3.11.4
./configure --enable-optimizations
make -j$(nproc)
sudo make altinstall
```

### Q: Node.js 安装失败？
A: 使用二进制包：
```bash
# 下载对应架构的 Node.js
wget https://nodejs.org/dist/v18.17.0/node-v18.17.0-linux-arm64.tar.xz
tar -xf node-v18.17.0-linux-arm64.tar.xz
sudo cp -r node-v18.17.0-linux-arm64/{bin,lib,share} /usr/local/
```

### Q: MySQL 安装失败？
A: 使用系统自带的 MariaDB：
```bash
sudo apt-get install mariadb-server
# 修改配置使用 MariaDB
# config.py 中 DATABASE_URL = "mysql+aiomysql://..."
```

## 性能优化建议

### 针对飞腾/鲲鹏 ARM 处理器

```python
# backend/app/main.py
import uvicorn

if __name__ == "__main__":
    # ARM 架构下使用单进程模式
    uvicorn.run(
        "app.main:app",
        host="0.0.0.0",
        port=8003,
        workers=1,  # ARM 架构建议单进程
        loop="uvloop"  # 高性能事件循环
    )
```

### 针对龙芯处理器

```bash
# 使用龙芯优化的 Python
export PYTHONOPTIMIZE=2

# 禁用某些不兼容的库
# requirements.txt 中注释掉：
# uvloop  # 龙芯不支持
```
