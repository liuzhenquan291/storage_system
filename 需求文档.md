# 仓储物资出入库动态管理系统需求文档

## 1. 项目概述

### 1.1 系统简介
仓储物资出入库动态管理系统是一个用于管理仓储物资出入库业务的综合性系统，支持库房管理、物资管理、工单管理、设备管理等核心功能。

### 1.2 技术栈
- **后端**: Python (Flask/FastAPI)
- **前端**: Node.js + Vue.js/React
- **数据库**: MySQL 8.0
- **部署方式**: Docker Compose
- **运行环境**: 单机内网访问

### 1.3 系统特点
- 单机运行，内网访问
- 无用户注册功能，由管理员统一管理用户
- 支持操作日志记录，所有出入库操作可追溯
- MySQL 数据持久化存储

## 2. 功能模块设计

### 2.1 用户权限管理模块

#### 2.1.1 用户管理
**功能描述**: 管理系统用户账号
- **用户列表**: 分页展示所有用户信息（用户名、姓名、角色、状态、创建时间等）
- **新增用户**: 
  - 必填字段：用户名、姓名、密码、角色、所属库房
  - 可选字段：联系电话、邮箱
  - 用户名唯一性校验
- **编辑用户**: 修改用户基本信息，不可修改用户名
- **删除用户**: 软删除，保留历史操作记录
- **重置密码**: 管理员可重置用户密码为默认密码
- **启用/禁用用户**: 控制用户账号状态

**数据表**: `sys_user`
```
- id: 主键
- username: 用户名（唯一）
- password: 密码（加密存储）
- real_name: 真实姓名
- role_id: 角色ID
- warehouse_id: 所属库房ID
- phone: 联系电话
- email: 邮箱
- status: 状态（0-禁用，1-启用）
- deleted: 删除标记（0-未删除，1-已删除）
- created_by: 创建人
- created_time: 创建时间
- updated_by: 更新人
- updated_time: 更新时间
```

#### 2.1.2 角色管理
**功能描述**: 管理系统角色及权限
- **角色列表**: 展示所有角色信息
- **新增角色**: 角色名称、描述、权限配置
- **编辑角色**: 修改角色信息和权限
- **删除角色**: 检查是否有用户使用，无则可删除
- **菜单权限配置**: 为角色分配菜单访问权限

**数据表**: `sys_role`
```
- id: 主键
- role_name: 角色名称
- role_code: 角色编码
- description: 描述
- deleted: 删除标记
- created_time: 创建时间
- updated_time: 更新时间
```

**数据表**: `sys_role_menu`
```
- id: 主键
- role_id: 角色ID
- menu_id: 菜单ID
```

#### 2.1.3 菜单管理
**功能描述**: 管理系统菜单结构
- **菜单列表**: 树形结构展示所有菜单
- **新增菜单**: 菜单名称、路径、图标、排序、父菜单
- **编辑菜单**: 修改菜单信息
- **删除菜单**: 检查是否有子菜单和角色关联

**数据表**: `sys_menu`
```
- id: 主键
- menu_name: 菜单名称
- menu_code: 菜单编码
- parent_id: 父菜单ID
- menu_path: 路由路径
- menu_icon: 图标
- menu_type: 类型（0-目录，1-菜单，2-按钮）
- sort_order: 排序
- status: 状态
- created_time: 创建时间
```

#### 2.1.4 系统初始化
**功能描述**: 系统首次启动时初始化管理员账号
- **配置文件**: `config/admin_config.yml`
  ```yaml
  admin:
    username: admin
    password: admin123
    real_name: 系统管理员
    role: 超级管理员
  ```
- **初始化流程**:
  1. 系统启动时检查数据库是否有管理员账号
  2. 无则从配置文件读取并创建
  3. 创建默认角色和菜单权限

```

```

### 2.4 工单管理模块

  - 展示所有入库工单

  - 更新库位状态

```
- remark: 备注
- deleted: 删除标记
- created_by: 创建人
- created_time: 创建时间
- updated_time: 更新时间
```

  - 支持按操作人、操作类型、时间范围筛选
- **操作类型**:
  - 登录/登出
  - 新增/编辑/删除
  - 入库/出库操作
  - 数据导入/导出

**数据表**: `sys_operation_log`
```
- id: 主键
- user_id: 操作用户ID
- username: 操作用户名
- operation_type: 操作类型
- operation_desc: 操作描述
- request_method: 请求方法
- request_url: 请求URL
- request_params: 请求参数
- response_result: 响应结果
- ip_address: IP地址
- operation_time: 操作时间
- created_time: 创建时间
```

#### 2.6.3 系统运行日志
**功能描述**: 记录系统运行状态
- **系统日志列表**:
  - 展示系统运行日志
  - 显示：日志级别、日志内容、时间
  - 支持按级别、时间范围筛选
- **日志级别**: INFO、WARN、ERROR

**数据表**: `sys_run_log`
```
- id: 主键
- log_level: 日志级别
- log_content: 日志内容
- log_source: 日志来源
- created_time: 创建时间
```

- **数据持久化**: Docker Volume

### 3.3 数据库设计原则
- 所有表包含主键 `id`（自增）
- 所有表包含 `created_time` 和 `updated_time`
- 逻辑删除使用 `deleted` 字段（0-未删除，1-已删除）
- 使用外键关联保证数据完整性
- 为常用查询字段创建索引

## 4. 部署方案

### 4.1 Docker Compose 配置
```yaml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: warehouse_mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: warehouse_db
      MYSQL_USER: warehouse
      MYSQL_PASSWORD: warehouse123
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    command: --default-authentication-plugin=mysql_native_password --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    networks:
      - warehouse_network

  # 后端服务
  backend:
    build: ./backend
    container_name: warehouse_backend
    restart: always
    environment:
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_NAME=warehouse_db
      - DB_USER=warehouse
      - DB_PASSWORD=warehouse123
      - JWT_SECRET=your-secret-key
      - ADMIN_USERNAME=admin
      - ADMIN_PASSWORD=admin123
    ports:
      - "5000:5000"
    depends_on:
      - mysql
    volumes:
      - ./backend:/app
      - ./logs:/app/logs
    networks:
      - warehouse_network

  # 前端服务
  frontend:
    build: ./frontend
    container_name: warehouse_frontend
    restart: always
    ports:
      - "3000:3000"
    depends_on:
      - backend
    volumes:
      - ./frontend:/app
    networks:
      - warehouse_network

  # Nginx 反向代理
  nginx:
    image: nginx:latest
    container_name: warehouse_nginx
    restart: always
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - frontend
      - backend
    networks:
      - warehouse_network

volumes:
  mysql_data:
    driver: local

networks:
  warehouse_network:
    driver: bridge
```

### 4.2 系统初始化流程
1. 启动 Docker Compose
2. MySQL 初始化数据库和表结构
3. 后端服务启动，检查管理员账号
4. 从环境变量读取管理员配置，创建默认账号
5. 创建默认角色和菜单权限
6. 前端服务启动
7. Nginx 配置反向代理
8. 系统就绪，可通过浏览器访问

### 4.3 数据持久化方案
- **MySQL 数据**: 使用 Docker Volume `mysql_data` 挂载到 `/var/lib/mysql`
- **应用日志**: 挂载宿主机 `./logs` 目录
- **配置文件**: 通过 Volume 挂载，方便修改
- **备份策略**: 
  - 定期执行 `mysqldump` 备份
  - 备份文件存储到挂载的宿主机目录

## 5. 安全设计

### 5.1 认证授权
- **认证方式**: JWT Token
- **Token 有效期**: 2小时
- **刷新机制**: Token 过期前30分钟可刷新
- **密码策略**: 
  - 最小长度6位
  - 使用 bcrypt 加密存储
  - 密码强度提示

### 5.2 权限控制
- **基于角色的访问控制 (RBAC)**
- **菜单权限**: 控制用户可访问的菜单
- **操作权限**: 控制用户的增删改查权限
- **数据权限**: 用户只能操作所属库房的数据

### 5.3 安全措施
- SQL 注入防护：使用 ORM 参数化查询
- XSS 防护：前端输入过滤，后端数据转义
- CSRF 防护：Token 验证
- 操作日志：记录所有敏感操作
- IP 白名单：可配置内网访问限制
