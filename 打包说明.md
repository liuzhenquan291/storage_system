# Windows 启动脚本使用说明

## 文件说明

| 文件 | 用途 |
|------|------|
| `start.sh` | Linux/Mac/Git Bash 启动脚本 |
| `start.bat` | Windows CMD/Batch 启动脚本 |

## 使用方法

### 方式一：直接使用 Batch 脚本（推荐）

1. 确保已安装 Docker Desktop (可参考 [docker安装教程.md](docker安装教程.md))
2. 双击运行 `start.bat`
3. 按提示进行配置
4. 自动启动服务

### 方式二：使用 Git Bash 运行 Shell 脚本

1. 安装 Git for Windows（包含 Git Bash）
2. 右键点击 `start.sh`，选择 "Git Bash Here"
3. 或者在 Git Bash 中运行：`bash start.sh`

### 方式三：打包成 EXE 文件

如果你希望将脚本打包成独立的 EXE 文件，可以使用以下工具：

#### 方法 1：使用 Bat To Exe Converter（简单）

1. 下载 Bat To Exe Converter：
   - 官网：https://www.f2ko.de/en/b2e.html
   - 或搜索 "Bat To Exe Converter v3.0.11"

2. 打开软件，选择 `start.bat` 文件

3. 配置选项：
   - **Version Information**：填写程序信息
     - Product Name: 仓储管理系统
     - Version: 1.0.0
   - **Options**：
     - ☑ Invisible（隐藏控制台窗口，推荐）
     - ☑ Encrypt the batch file（加密脚本）
     - ☑ Administrator manifest（需要管理员权限）
   - **Include**：添加依赖文件
     - `docker-compose.yml`
     - `backend/` 文件夹
     - `frontend/` 文件夹
     - `docker/` 文件夹

4. 点击 "Compile" 生成 `start.exe`

#### 方法 2：使用 Inno Setup（专业安装包）

1. 下载安装 Inno Setup：
   - 官网：https://jrsoftware.org/isinfo.php

2. 创建脚本文件 `setup.iss`：

3. 使用 Inno Setup 编译生成安装程序

#### 方法 3：使用 Advanced Installer（商业软件）

适合企业级部署，可创建专业的 MSI 安装包。

## 脚本功能说明

### 启动脚本会执行以下步骤：

1. **检查 Docker**
   - 检查 Docker 是否安装
   - 检查 Docker Compose 是否安装
   - 检查 Docker 是否正在运行
   - 如果未安装，提示安装方法

2. **选择数据路径**
   - 默认路径：`%USERPROFILE%\warehouse_data`
   - 支持自定义路径
   - 自动创建目录

3. **配置系统参数**（交互式）
   - MySQL ROOT 密码
   - 数据库普通用户名/密码
   - 管理员账号/密码/昵称
   - 自动生成 JWT 密钥

4. **生成 .env 文件**
   - 自动填充配置
   - 端口使用默认值
   - 数据库配置前后端一致

5. **启动服务**
   - 拉取 Docker 镜像
   - 构建并启动容器
   - 显示访问地址

### 默认端口配置

| 服务 | 端口 |
|------|------|
| MySQL | 3307 |
| 前端 | 3003 |
| 后端 | 8003 |
| Nginx | 80 |

## 注意事项

1. **首次运行需要联网**，用于拉取 Docker 镜像
2. **数据持久化**：数据库数据存储在用户选择的数据路径中
3. **管理员账号**：系统初始化时创建，后续可通过界面管理
4. **Windows 防火墙**：首次运行可能需要允许 Docker 通过防火墙

## 常见问题

### Q: 提示 "Docker 未安装" 但已安装 Docker Desktop
A: 确保 Docker Desktop 已启动，并且已在设置中启用 "Use the WSL 2 based engine"

### Q: 中文显示乱码
A: 脚本已使用 UTF-8 编码，确保 Windows 区域设置中已启用 "Beta: Use Unicode UTF-8 for worldwide language support"

### Q: 如何修改配置？
A: 删除 `.env` 文件，重新运行启动脚本即可重新配置

### Q: 如何备份数据？
A: 备份数据目录（默认 `%USERPROFILE%\warehouse_data`）和 `.env` 文件
